To build fast, we will set up a single clean relational database (covid_system).  1. Database Schema (schema.sql)users table: Handles authentication for Patients and Admins.  id, name, email, password, phone, address, location, role ('admin', 'patient'), created_at.  hospitals table: Handles hospital-specific registry and approval states.  id, hospital_name, email, password, address, location, status ('pending', 'approved', 'rejected'), created_at.  vaccines table: Tracks inventory visibility.  id, vaccine_name, status ('available', 'unavailable').  appointments table: Core transactional hub connecting patients, hospitals, actions, and results.  id, patient_id, hospital_id, type ('test', 'vaccination') , appointment_date, status ('pending', 'approved', 'rejected') , test_result ('pending', 'negative', 'positive') , vaccination_status ('not_vaccinated', 'vaccinated').  2.
